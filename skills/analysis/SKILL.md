# Stage 5 — Analysis

Goal: every RQ gets its figure or table, generated by script from the data in
`paper/experiments/`, ready for the manuscript.

## Figures

One Python script per figure in `paper/figures/`, executed with the
`render_figure` MCP tool. Scripts read from `experiments/*/data.csv` —
**never** from numbers pasted into the script; the figure must regenerate if
the experiment reruns.

After each render, Read the produced PDF and look at it — legend placement,
label sizes, clipping — before wiring it into the manuscript. (Drawn figures
— architecture, state, flow diagrams — are not data figures; they are covered
by the writing skill's references/diagrams.md.)

Conventions that make figures read as publication-grade:

- **Form follows the question.** Trends over a variable → line plot.
  Cross-system comparison → grouped bars. Latency distributions → CDFs
  (the systems staple). Never a bar chart for a distribution.
- **Vector PDF output**, fonts embedded, sized so labels match the paper's
  body text when placed at `\columnwidth` (set the figure size to the final
  physical size — roughly 3.3in wide for one column — instead of scaling down
  a large figure).
- **Error bars or shaded CI bands always**, from the repeated runs. A mean
  without variance is a red flag to reviewers.
- Colorblind-safe palette; additionally vary marker/linestyle so the figure
  survives grayscale printing. Legend inside the axes when there is room.
- Log axes where the data spans decades — and label them as such.
- Normalize where it clarifies (speedup over baseline = 1.0) and use
  geometric mean for normalized results across benchmarks.

## Tables

- `booktabs` only: `\toprule`/`\midrule`/`\bottomrule`, no vertical rules.
- Numbers right-aligned, consistent precision, units in the header.
- Bold the best result per column. Every table needs a "read this row first"
  design — reviewers scan, they do not study.
- Standard tables to prepare: experimental-setup summary (hardware, software
  versions, workloads) and the headline comparison table.

## Takeaways

Update `experiments/results.md`: under each RQ, the final figure/table
reference and a one-sentence finding phrased the way it will appear in the
paper ("X sustains 2.3× the throughput of Y beyond 64 clients"). These
sentences become the boxed takeaways of the Evaluation section.

Also nominate a **headline figure** — the single plot that makes the paper's
case — as a candidate for Figure 1.

No gate here: flow directly into writing, but keep STATE.md current.
